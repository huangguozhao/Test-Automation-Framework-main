# Test Report Viewer Guide

## Overview
This tool helps you view and share test history reports generated by the test automation framework.

## How to Use

### 1. Start the Report Viewer
Run the following command in your terminal:
```bash
python view_report.py
```

### 2. Available Options

#### Option 1: List All History Reports
- Displays all available historical test reports
- Reports are sorted by date (newest first)
- Format: `easyjava_YYYYMMDD_HHMMSS`

#### Option 2: View a Report (Live Server)
- Opens the selected report using Allure's live server
- Allows interactive exploration of test results
- Press `Ctrl+C` to stop the server
- **Best for**: Local viewing and debugging

#### Option 3: Generate Static Report for Sharing
- Creates a standalone HTML report
- Can be shared with others without running a server
- **Best for**: Sharing with team members or stakeholders

#### Option 4: Exit
- Closes the report viewer tool

## Examples

### Example 1: View the Latest Report
1. Run `python view_report.py`
2. Select option `2`
3. Leave the report name empty and press Enter
4. Select the first report (usually the latest)

### Example 2: Share a Report with Others
1. Run `python view_report.py`
2. Select option `3`
3. Select the report you want to share
4. Wait for the static report to generate
5. Navigate to `./report/static/{report_name}/`
6. Share the entire folder with others
7. Recipients can open `index.html` to view the report

### Example 3: View a Specific Report
1. Run `python view_report.py`
2. Select option `2`
3. Enter the exact report name (e.g., `easyjava_20260103_080007`)
4. The report will open in your browser

## Report Locations

### Raw Reports (Allure Data)
- Location: `./report/easyjava_YYYYMMDD_HHMMSS/`
- Contains: JSON files, attachments, test results
- Used by: Allure to generate reports

### Static Reports (HTML)
- Location: `./report/static/easyjava_YYYYMMDD_HHMMSS/`
- Contains: HTML, CSS, JavaScript files
- Used by: Direct viewing in browser, sharing

## Tips

1. **For Quick Viewing**: Use option 2 (live server) for interactive exploration
2. **For Sharing**: Use option 3 (static report) to create portable reports
3. **Report Naming**: Reports are named with timestamps for easy identification
4. **Storage**: All historical reports are preserved by default

## Troubleshooting

### Allure Command Not Found
- Ensure Allure is installed: https://docs.qameta.io/allure/
- Add Allure to your system PATH
- Verify installation: `allure --version`

### Report Not Opening
- Check if the report directory exists
- Ensure the report contains valid Allure data files
- Try generating a static report instead

### Browser Not Opening
- Check your default browser settings
- Manually open `index.html` in your browser
- Try a different browser

## Requirements

- Python 3.6+
- Allure Command-line (installed and in PATH)
- Modern web browser (Chrome, Firefox, Edge, etc.)
